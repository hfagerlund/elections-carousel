Release notes

Links to all release notes are also available at https://github.com/hfagerlund/elections-carousel/tags.

  • Related reference: CHANGELOG

v1.0.0

(Apr 26, 2017)

New Features:

  • Launched project documentation site
  • New architecture (includes UMD support)
  • Client-side templating (Mustache)
  • Added custom script to automate updating of dynamic chunkhash values from (CSS/JS) bundler in demo page
  • Added .editorconfig
  • Integrated with Travis
  • CHANGELOG separated from README

Changed:

  • Modified build toolchain (added Webpack; removed Grunt, Bower)
  • Replaced QUnit test harness with TAPE (to support Node as well as browser)
  • Revised CSS naming conventions
  • Modified JS linting (replaced JSHint with ESLint)

Fixed:

  • Accessibility compliance, keyboard support for button controls
  • Button controls 'jumping' on press/release bug

v0.4.0

(Jan 13, 2016)

New Features:

  • New look-and-feel (GUI improvements)
  • switched from horizontal to vertical bars
    • height of bars provides visualization of percentage of votes per candidate
  • styling improvements: font updates, gradients, party 'labels' (included in list format), emphasized riding vote totals
  • (minor) text copy changes

Additional changes:

  • improved alphabetical ordering of styling rules
  • updated copyrights
  • removed unnecessary role attributes
  • bumped version number
  • updated README (Changelog, screenshot)

v0.3.0

(Jan 13, 2016)

New Features:

  • Reordered results: candidates within each riding now listed in descending order by number of votes

v0.2.1

(Dec 14, 2015)

Fixed:

  • Removed deprecated code

v0.2.0

(Dec 14, 2015)

New features:

  • Defined global, read-only variables (JSHint strict mode)
  • Automated package management

Changed:

  • Refactored code, performance improvements
  • Improved vote percentages (to 2 decimal places; totals 100%)
  • Accessibility enhancement: 'Elected' text rendering

Fixed:

  • Button (enabled/disabled) state bugs

v0.1.1

(Jul 5, 2015)

New features:

  • Enhanced button states

New features:
